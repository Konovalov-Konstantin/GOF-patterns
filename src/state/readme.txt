
                                            Паттерн СОСТОЯНИЕ.

Паттерн Состояние управляет изменением поведения объекта при изменении его внутреннего состояния. Внешне это выглядит
так, словно объект меняет свой класс.

Преимущества паттерна:
- Локализация поведения каждого состояния в отдельном классе.
- Исключение многочисленных конструкций if, усложнявших сопровождение кода (см. класс GumballMachine).
- Каждое состояние закрыто для изменения, однако сам автомат открыт для расширения посредством добавления новых классов состояний.
- Создание кодовой базы и структуры классов, приближенной к диаграмме переходов автомата, а следовательно, более простой для чтения и понимания.

В классе GumballMachine_V2 хранится экземпляр каждого класса состояния. Текущее состояние автомата всегда представлено одним из этих экземпляров.
Действия делегируются объекту текущего состояния.
